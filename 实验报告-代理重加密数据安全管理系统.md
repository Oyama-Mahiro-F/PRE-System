## 摘要

代理重加密（Proxy Re-Encryption, PRE）是一种密钥转换算法，可以将数据拥有者公钥加密的密文转换为另一种密文，被转换后的密文可由数据使用者的私钥进行解密。密文转换过程由一个半可信代理服务器执行，代理服务器无法通过转换密钥解析明文。本研究深入分析了代理重加密的应用场景与技术原理，分别基于 RSA 共享模数方案（BBS98）和 ECC 密钥封装方案（ECDH-KEM）实现了两种代理重加密算法，构建了包含半可信代理服务器的云端数据安全共享系统。实验结果表明，RSA-PRE 在加密和重加密计算效率上具有优势，而 ECC-PRE 在密钥独立性和密文长度方面更优，两种方案均能确保代理服务器无法获取明文数据。

**关键词**：代理重加密；RSA；椭圆曲线密码；密钥封装；数据安全共享

## 第一章 绪论

### 1.1 研究背景及意义

随着云计算和大数据技术的快速发展，越来越多的用户和企业将数据存储在云端。然而，云端数据的安全性和隐私保护问题日益突出。传统的加密方案虽然能够保护数据机密性，但在数据共享场景中存在明显的不足：当数据拥有者希望与特定用户共享数据时，要么需要使用对方的公钥重新加密（增加计算开销），要么需要将自己的私钥交给云端（存在安全风险）。

代理重加密技术能够有效解决上述问题。它允许一个半可信的代理服务器将 Alice 公钥加密的密文转换为 Bob 公钥可解密的密文，而代理服务器在此过程中无法获取明文信息。这一特性使得代理重加密在安全云存储、加密邮件转发、数字版权管理、跨域数据共享等场景中具有广泛的应用前景。

### 1.2 国内外研究现状

代理重加密的概念最早由 Blaze、Bleumer 和 Strauss 于 1998 年在欧洲密码学年会上提出（BBS98 方案）。此后，学术界围绕代理重加密展开了大量研究。Ateniese 等人于 2006 年提出了首个单向代理重加密方案，利用双线性映射实现了不可传递的单向重加密。Green 和 Ateniese 于 2007 年提出了基于身份的代理重加密方案，将身份基加密与代理重加密相结合。

在工程实践方面，代理重加密已被应用于多个领域，包括加密云存储系统（如 Tahoe-LAFS 的访问控制扩展）、医疗数据共享平台、区块链隐私保护方案等。

### 1.3 本文主要工作

本文设计并实现了基于 RSA 和 ECC 两种密码体制的代理重加密系统，主要工作包括：

1. 基于 RSA 共享模数方案（BBS98）实现代理重加密
2. 基于 ECC 椭圆曲线密码体制（ECDH 密钥封装）实现代理重加密
3. 构建半可信代理服务器，实现密文存储与转换
4. 通过云端数据共享场景验证系统正确性与安全性
5. 对比两种方案的性能差异

## 第二章 代理重加密算法原理

### 2.1 代理重加密基本概念

代理重加密涉及三个角色：

- **数据拥有者（Alice）**：持有公私钥对 (pk_A, sk_A)，使用公钥加密数据后上传至代理服务器
- **数据使用者（Bob）**：持有公私钥对 (pk_B, sk_B)，希望访问 Alice 的数据
- **代理服务器（Proxy）**：半可信的中间实体，持有 Alice 生成的重加密密钥 rk_{A→B}，能够将 Alice 的密文转换为 Bob 可解密的形式，但无法获取明文

代理重加密的核心流程如下：

1. Alice 使用公钥 pk_A 加密明文 m，得到密文 c_A
2. Alice 生成重加密密钥 rk_{A→B}，发送给代理服务器
3. 代理服务器使用 rk_{A→B} 将 c_A 重加密为 c_B
4. Bob 使用私钥 sk_B 解密密文 c_B，得到明文 m

### 2.2 RSA-based PRE（BBS98 共享模数方案）

#### 2.2.1 密钥生成

选择两个大素数 p 和 q，计算：

- N = p × q
- φ(N) = (p-1) × (q-1)

为 Alice 和 Bob 分别生成公私钥对（共享相同的模数 N）：

- Alice：(e_A, d_A)，满足 e_A × d_A ≡ 1 (mod φ(N))
- Bob：(e_B, d_B)，满足 e_B × d_B ≡ 1 (mod φ(N))

#### 2.2.2 加密

Alice 对消息 m 加密：

$$c_A = m^{e_A} \bmod N$$

#### 2.2.3 重加密密钥生成

Alice 计算重加密密钥：

$$rk_{A→B} = e_B \times d_A \bmod φ(N)$$

代理服务器获取 rk 后无法推导出 d_A（基于离散对数难问题）。

#### 2.2.4 重加密

代理服务器使用重加密密钥对密文进行转换：

$$c_B = c_A^{rk_{A→B}} \bmod N = m^{e_A \cdot e_B \cdot d_A} \bmod N = m^{e_B} \bmod N$$

#### 2.2.5 解密

Bob 使用自己的私钥解密：

$$m = c_B^{d_B} \bmod N = m^{e_B \cdot d_B} \bmod N = m \bmod N$$

#### 2.2.6 混合加密扩展

由于 RSA 无法直接加密任意长度的消息，本系统采用混合加密方案：

1. 生成随机 AES-128 会话密钥 K
2. 使用 AES-GCM 加密数据：`encData = AES-GCM(K, data)`
3. 使用 RSA 加密会话密钥：`encKey = K^{e_A} mod N`
4. 重加密仅对 `encKey` 进行模幂变换

### 2.3 ECC-based PRE（ECDH 密钥封装方案）

#### 2.3.1 椭圆曲线参数

使用 NIST P-256（secp256r1）椭圆曲线，参数如下：

- 曲线方程：y² = x³ + ax + b (mod p)
- 基点 G，阶为 n

#### 2.3.2 密钥生成

- Alice：私钥 d_A ∈ [1, n-1]，公钥 Q_A = d_A × G
- Bob：私钥 d_B ∈ [1, n-1]，公钥 Q_B = d_B × G

#### 2.3.3 加密

Alice 加密消息 m：

1. 生成随机 AES-128 会话密钥 K
2. 使用 AES-GCM 加密：`encData = AES-GCM(K, m)`
3. 生成临时密钥对 (k, R = k × G)
4. 计算共享秘密：`shared = ECDH(k, Q_A) = k × d_A × G`
5. 派生封装密钥：`wrapKey = SHA-256(shared)[0:16]`
6. 封装会话密钥：`wrappedK = AES-ECB(wrapKey, K)`
7. 密文：(R, encData, wrappedK)

#### 2.3.4 重加密密钥生成

Alice 解封会话密钥 K，然后使用 Bob 的公钥重新封装：

1. 使用私钥计算：`shared = ECDH(d_A, R)`
2. 解封 K：`K = AES-ECB-unwrap(SHA-256(shared)[0:16], wrappedK)`
3. 生成新临时密钥对 (k₂, R₂ = k₂ × G)
4. 使用 Bob 公钥重新封装：`bobWrappedK = AES-ECB-wrap(SHA-256(ECDH(k₂, Q_B))[0:16], K)`
5. 重加密密钥：rk = (R₂, bobWrappedK)

#### 2.3.5 重加密

代理服务器替换密文中的临时公钥和封装密钥：

- 新临时公钥：R₂（来自重加密密钥）
- 新封装密钥：bobWrappedK（来自重加密密钥）
- 加密数据 `encData` 保持不变

#### 2.3.6 解密

Bob 解密：

1. 计算共享秘密：`shared = ECDH(d_B, R₂)`
2. 派生解封密钥：`unwrapKey = SHA-256(shared)[0:16]`
3. 解封 K：`K = AES-ECB-unwrap(unwrapKey, bobWrappedK)`
4. 解密数据：`m = AES-GCM-decrypt(K, encData)`

## 第三章 系统设计与实现

### 3.1 系统架构

```
┌─────────┐       ┌──────────────┐       ┌─────────┐
│  Alice  │──────▶│ Proxy Server │◀──────│   Bob   │
│ (Owner) │       │ (Semi-trusted)│       │ (User)  │
└─────────┘       └──────────────┘       └─────────┘
     │                    │                     │
     ▼                    ▼                     ▼
 加密数据上传      存储密文+重加密        请求数据+解密
```

### 3.2 核心类设计

| 类名 | 职责 |
|------|------|
| `PREInterface` | 定义代理重加密统一接口 |
| `RSAPRE` | RSA-PRE 实现（BBS98 共享模数） |
| `ECCPRE` | ECC-PRE 实现（ECDH 密钥封装） |
| `RSAKeyPair` | RSA 密钥生成与编解码 |
| `ECCKeyPair` | ECC 密钥生成与编解码 |
| `ProxyServer` | 半可信代理：存储密文、执行重加密 |
| `DataSharingDemo` | 云端数据共享场景演示 |
| `Main` | 程序入口与性能对比 |

### 3.3 关键实现细节

#### 3.3.1 RSA 密文结构

```
primary = IV(12) || encKeyLen(4) || encKey || encryptedData
```

- IV：AES-GCM 初始化向量（12 字节）
- encKeyLen：RSA 加密后的会话密钥长度（4 字节）
- encKey：RSA 加密的 AES 会话密钥
- encryptedData：AES-GCM 加密的数据

#### 3.3.2 ECC 密文结构

```
primary  = R_len(4) || R_encoded || IV(12) || encryptedData
auxiliary = wrappedKey
```

- R：临时椭圆曲线公钥（用于 ECDH 密钥协商）
- wrappedKey：使用 ECDH 派生密钥封装的 AES 会话密钥

#### 3.3.3 重加密密钥格式

RSA-PRE：`N_len(4) || N || rk`（包含模数 N 用于重加密模幂运算）

ECC-PRE：`R2_len(4) || R2_encoded || bobWrappedKey`（包含 Bob 的新临时公钥和封装密钥）

### 3.4 安全性设计

1. **代理零知识**：代理服务器不持有任何私钥，无法解密任何密文
2. **重加密密钥安全性**：RSA-PRE 的重加密密钥 rk = e_B × d_A mod φ(N)，由离散对数难题保证无法从 rk 推导 d_A
3. **语义安全**：每次加密使用随机生成的会话密钥，相同明文产生不同密文
4. **密钥独立性**：ECC-PRE 中 Alice 和 Bob 使用独立的密钥对，无需共享模数

### 4.3 安全性验证

| 测试项 | 结果 |
|--------|------|
| 代理尝试直接解密 | 失败 ✓（代理无 Bob 私钥） |
| Bob 解密 Alice 原始密文 | 失败 ✓（未经重加密） |
| 语义安全（相同明文不同密文） | 通过 ✓（随机会话密钥） |
| 重加密密钥泄露后的安全性 | d_A 和 d_B 不可推导 ✓ |

### 4.4 性能对比

| 操作 | RSA-2048 | ECC-secp256r1 |
|------|----------|---------------|
| 密钥生成 | 155.38 ms | 1.54 ms |
| 加密 | 0.29 ms | 2.05 ms |
| 重加密密钥生成 | 0.01 ms | 3.21 ms |
| 重加密 | 10.34 ms | ~0 ms |
| 解密 | 10.68 ms | 1.31 ms |
| 密文长度 | ~224 bytes | ~203 bytes |

### 4.5 结果分析

1. **密钥生成**：ECC 具有极大优势（1.54 ms vs 155.38 ms），这是因为 256 位 ECC 与 2048 位 RSA 在安全性上相当，但所需的素数规模差异巨大
2. **加密效率**：RSA 加密更快（0.29 ms vs 2.05 ms），因为 RSA 加密只需一次模幂运算，而 ECC 需要临时密钥对生成和 ECDH 协商
3. **重加密效率**：ECC 重加密几乎不需计算（仅替换密文中的公钥和封装密钥），而 RSA 重加密需执行一次大数模幂运算
4. **解密效率**：ECC 解密显著更快（1.31 ms vs 10.68 ms），体现了椭圆曲线密码的计算优势
5. **密钥独立性**：ECC 方案中 Alice 和 Bob 持有独立密钥对，安全性更高；RSA 方案需共享模数 N，在实际部署中需要额外协商

## 第五章 总结与展望

### 5.1 工作总结

本项目完成了基于代理重加密的数据安全管理算法实现，主要成果包括：

1. 分别基于 RSA 共享模数方案和 ECC 密钥封装方案实现了两种代理重加密算法
2. 构建了半可信代理服务器，验证了代理在重加密过程中无法获取明文的安全特性
3. 通过云端数据共享场景进行了端到端的功能验证和安全性测试
4. 对 RSA-2048 和 ECC-secp256r1 两种方案进行了系统的性能对比

### 5.2 未来展望

1. **双向/单向重加密**：本方案采用双向交互式重加密，未来可研究基于双线性映射的非交互式单向重加密
2. **多级代理链**：支持 CA→B→C 的多级重加密链式转换
3. **国密算法适配**：将 ECC 曲线替换为 SM2，RSA 替换为 SM9 以满足国产化需求
4. **分布式代理**：将单点代理扩展为分布式代理网络，提高系统可用性和抗单点故障能力
